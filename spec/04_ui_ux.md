# REFLEX LABORATORY仕様書 - 04. UI/UX・デザイン仕様

## 4.1 デザインコンセプト
- **「Cyberpunk / Clinical Laboratory」スタイル**
  - ダークトーン（背景色：`zinc-950`）を基調とします。
  - アクセントカラーとして、未来的な「高輝度シアン（Cyan-400 / 500）」と「アンバー（Amber-400）」をブレンドします。
  - 統計情報やデータ部分には `JetBrains Mono` または `Fira Code` などの等幅フォントを用い、実験的な計測器としての精密な印象を演出します。
  - 余白を広く持たせた無機質でミニマルなレイアウトを採用します。

## 4.2 主要インターフェース構成
1. **ナビゲーションヘッダー**  
   - タイトル「REFLEX LABORATORY / 反応速度測定ラボ」  
   - 実験のシリアルコードを模したセッションID（例:`LAB-502-C`）を自動生成して表示。
2. **キャリブレーション・コントロール部**  
   - 現在のモニター設定。リフレッシュレート自動検出の結果や、手動プルダウンを備えます。
   - 「基準値一覧」をアコーディオン方式で閉じたり開いたりできる機能を持ちます。
3. **測定サークルエリア (Reaction Arena)**  
   - 画面の中央に配置される巨大な角丸のコンテナ。
   - クリック/タップ判定が即時に PointerDown で検知できるよう全面がインタラクティブになっています。
   - 待機（WAITING）時は、余分な円の表示をすべて非表示とし、画面中央に視線を集中させるための小さなドット（点）のみを点滅させずに常時静かに表示します（チカチカする動きを排除して最高度の集中を促します）。
   - 反応検出（ACTIVE）時は、目印のドットが消えると同時に、設定よりプレイヤーが指定した「円の色」（シアン、緑、ピンク、黄色、白の5色）のサークル（「クリック！」）が突如として出現します。余計な波動（波紋）や広がるエフェクトを完全に削ぎ落とし、円がシンプルに現れるだけの設計としています。
4. **クイック統計グリッド**  
   - これまでの合計テスト回数。
   - トラックしてきた平均反応速度（ms）。
   - 歴代の自己ベストスコアを表示。
5. **測定推移グラフ（履歴が2件以上の場合に表示）**  
   - CSSまたはインラインSVGによるカスタムベクター線グラフ。
   - 反応速度（ms）は数値が小さいほど「優れた高速反応（改善）」となるため、縦軸は「上に行くほど速い（数値が小さい）」ように定義。これにより、練習を続けるほど線が右肩上がりに上昇し、改善の成果が直感的に視覚化されます。
   - 縦軸のラベルには、一番上に「○○ms (速い)」、一番下に「○○ms (遅い)」を明記し、視覚的な基準を明確化します。
   - 横軸は日付（date）ではなく、記録順に「1回目、2回目…」と等間隔で配置。
   - 各データポイント（円）にカーソルを合わせると、何回目の測定か、測定時間、および測定日時（"6/17 16:43"形式の文字列）が2行のツールチップとして動的に表示されます。
6. **別タブで直接プレイ（拡張プレイ機能）**
   - iframe内やブラウザ拡張機能等によるラグ・フォーカスの影響から逃れ、より厳密に測定を行いたいプレイヤー向けに、別の独立したタブでゲームを起動するリンク（「別タブで直接プレイ」）を設置。
   - (※過度な重複を避け、シンプルさを維持するため、過去に存在したスタンドアロン型HTMLダウンロード機能は非搭載とし、別タブプレイに機能をシンプルに一本化します)
7. **履歴クリアと確認ダイアログ**
   - 保存された、すべての過去履歴レコードを全消去するための「履歴クリア」ボタンを履歴リストの上部または近傍に設置。
   - 誤操作による重要な測定データの消失を防ぐためのセキュリティガードとして、クリアボタン押下時に「すべての実験履歴データを初期化しますか？（この操作は取り消せません）」といった確認モーダル/ダイアログを表示する。
8. **SNS共有用のスコアコピー機能**
   - 5ラウンドの測定がすべて完了した際の結果（RESULT）画面に、ワンタップで測定結果をクリップボードにコピーできる「結果をコピー」ボタンを設置する。
   - コピーされるテキストフォーマット例：
     ```text
     🔬 REFLEX LABORATORY 反応速度測定結果
     平均反応速度: [平均スコア] ms
     評価ランク: [評価名]
     測定環境: [設定Hz] Hz モニター
     測定セッション: [セッションID]
     #ReflexLaboratory #反応速度測定 [App URL]
     ```
   - コピー成功時、ボタンのラベル表記が一時的に「コピー完了！」に切り替わり、チェックマーク等に変化するマイクロインタラクションフィードバックを設ける。

9. **円のカスタマイズ機能（色の選択）**
   - 設定エリア（基準モニター設定の直下）に「外観」セクションおよび「円の色設定」UIを新設し、円が出現したとき（クリックしてよい瞬間）の色を、**シアン、緑、ピンク、黄色、白**の5つのカラーパレットから動的に変更可能とする。
   - 変更された選択色は、`localStorage`（キー：`reaction_tester_circlecolor`）に保存し、ブラウザをリロードした際や再訪問時にも選択状態を自動維持する。
   - ※待ち時間中の視線固定ドットの色は、余計な誤読やお手つきを防止及び集中力強化のため、常に落ち着いた固定のドット色（グレー）に固定されます。

10. **コード生成による効果音（音響フィードバック）**
    - **効果音設定 (オン/オフ)**: 設定エリアの「音響」セクションにあるトグルスイッチから、すべての効果音を一瞬でオン/オフ（無音）に切り替え可能。設定値は `localStorage`（キー：`reaction_tester_soundenabled`）に保存され、ブラウザのリロード後や次回訪問時にも自動適用・維持されます。
    - ゲーム内の極めて繊細なタイミングに合わせ、**Web Audio API を用いて完全な手触りを持つシンセサイザー合成音をブラウザ側でリアルタイム生成**。
    - **出現音**: 視覚への純粋な反応測定を阻害しないよう廃止（消音）。
    - **成功音 ("ポン")**: ピッチ550Hz〜400Hzスイープ (Sine波) で約180ms、クリック完了時に軽い快感フィードバック。
    - **警告音 ("ブー")**: 太めの不協和140Hz+143Hzデチューン (Sawtooth波の2連) で約350ms、早まったクリック（フライング）への明確な警告。
    - 音声ファイル読み込み時間（物理DL遅延）やアセットエラーの可能性を完全に排除した、極致のレスポンス設計。音量は常時控えめかつ上品にマージされています。

 11. **ツールの目的提示メッセージ（スタート画面）**
     - **表示文言**: 「FPSのウォームアップに。反応速度は鍛えると速くなる」
     - **配置**: スタート画面（計測開始前の `IDLE` 状態）において、計測を開始するための巨大な「測定を開始する (5ラウンド)」ボタンのすぐ真上に配置。
     - **デザインスタイル**: サイズは `11px`〜`12px` 程度。フォントカラーには控えめな高輝度シアン（例: `cyan-400/80`）または上品なライトグレー（例: `zinc-400`）を使用し、他の要素と調和するよう、過度な主張を抑えつつも視認性を確保した上品なデザイン。
     - **挙動**: 計測中（WAITING, ACTIVE, FOUL, ROUND_RESULTなど）や最終結果（RESULT）画面では表示されず、最初の「スタート画面（IDLE）」でのみ表示されます。
