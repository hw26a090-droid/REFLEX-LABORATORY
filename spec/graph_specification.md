# REFLEX LABORATORY仕様書 - 計測モード推移グラフの強化仕様

本仕様書は、計測モード（記録が保存される本番モード）における履歴グラフ画面の機能強化および視覚的改善に関する仕様を定義します。
「継続すると速くなる＝役に立つ」という体験を視覚的に強化し、モチベーションの向上と直観的な変化の把握を支援することを目的とします。

---

## 1. 強化の目的
- **成長の可視化**: 自己ベストの達成や、自身の平均タイムの変化をより直観的に把握できるようにします。
- **直近の変化の理解**: 前回の計測結果と比較してどれだけ改善したか、速くなったか（反応速度の向上）を明確に伝えます。
- **情報の整理**: 従来の「中央値（Median）」を示す補助線を廃止し、「平均（Average）」に一本化することで、グラフ画面の複雑度を下げて洗練されたミニマルなデザインを維持します。

---

## 2. 強化機能詳細

### ① 自己ベスト（最速記録）の強調
グラフ上の点のうち、最も速いタイム（ミリ秒数値が最小）の計測ポイントを、他のポイントと区別して目立たせます。

- **視覚的表現**:
  - **ポイント（点）の色**: ゴールド（Tailwind: `amber-500` / `#f59e0b`）に変更。
  - **ラベル**: ポイントのすぐ上に「**★ BEST**」という小さなゴールド色のラベルを表示。
- **タイ記録時の挙動**:
  - 最速タイム（自己ベスト）が複数存在する場合は、**最も日付が新しい（右側の）1点のみ**を強調表示します。
- **仕様の意図**:
  - 過去のピークパフォーマンスを常にグラフ上で意識させ、「この記録を超えたい」というモチベーションを生み出します。

### ② 平均ラインの表示（中央値ラインの廃止）
すべての測定記録の平均タイムを示す水平線をグラフ内に配置します。

- **視覚的表現**:
  - **スタイル**: 点線（破線）、控えめなグレー系（Tailwind: `zinc-700/80` など）で、本体のシアン色の折れ線と明確に区別。
  - **ラベル**: 線の右端または空きスペースに、控えめなテキストで「**平均: ○ms**」と表示。
- **既存仕様からの変更点**:
  - 現状のグラフで表示されている「中央値（Median）」の補助線および「○ms（中央値）」のラベルは、認知負荷を避けるため**完全に廃止**し、この「平均ライン」へと置き換えます。

### ③ 前回比の差分表示
最新（一番右側）の記録が、その直前の記録（右から2番目）と比べて、どれだけ反応速度が変化したかを1行で表示します。

- **配置位置**:
  - グラフタイトルの直下、かつグラフ描画エリアの真上。独立した1行のテキストエリア。
- **表記およびカラー仕様**:
  - **速くなった場合（タイムが減少）**:
    - 表記例：「**前回比 -5ms ↑**」
    - カラー：シアン系（Tailwind: `cyan-400` / `#22d3ee`）
  - **遅くなった場合（タイムが増加）**:
    - 表記例：「**前回比 +8ms ↓**」
    - カラー：控えめなグレー系（Tailwind: `zinc-500` / `#71717a`）
  - **変化がない場合（タイムが同等）**:
    - 表記例：「**前回比 ±0ms**」
    - カラー：控えめなグレー系（Tailwind: `zinc-500` / `#71717a`）
- **表示条件**:
  - 保存された記録が**1件のみ**で、比較対象となる「前回」の記録が存在しない場合は、この領域自体を**非表示**（またはスペースを詰めない非表示状態）にします。

## 3. UI/UX配置イメージ

```text
+-------------------------------------------------------------+
|  【過去の測定結果の推移】                                    |
|   前回比 -5ms ↑                                (シアン色表示)  |
|                                                             |
|   (タイム: ms)                                              |
|    ▲ (速い)                                                 |
|    |               ★ BEST (ゴールド)                        |
|    |                 ● [210ms]                              |
|    |                / \                                     |
|    |   ● [230ms]   /   \                                    |
|    |    \         /     ● [225ms]                           |
|  - - - - \ - - - ● [220ms] - - - - - - - - - - - -  平均: 221ms (破線: グレー)
|    |      \     /                                           |
|    |       ● [235ms]                                        |
|    +---------------------------------------------------->   |
|                                                  (測定回数) |
+-------------------------------------------------------------+
```

## 4. 技術的な前提・制約
- **プラットフォーム互換性**: Vite + React（TypeScript）構成。ビルド時に `vite-plugin-singlefile` を使用することで、CSSスタイルやJS、SVGアイコンを含むすべての依存関係を1つのHTMLファイルにインライン化します。これにより、React製アプリでありながらオフライン環境・外部CDN依存なしで完結して動作する `reflex-test-standalone.html` が自動生成されます。
- **データ永続化**: すべての記録履歴は `localStorage` より取得します（既存のロードロジックをそのまま活用）。
- **リフレッシュレート補正の適用**: グラフ上の各点は、リフレッシュレート補正適用後のタイム値を使用します。
